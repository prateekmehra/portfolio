Chronos

I wrote a high risk, scalable scheduler service capable of authorizing and responding to client applications requests for scheduling tasks by hitting back a URL with specified methods, headers, and payloads. Tasks could be scheduled to be recurring or non-recurring depending on the requirement of the client.

APIs:
1. GET /get-jobs
2. POST/PUT /schedule-job
3. POST/PUT /reschedule-job
4. POST/PUT /unschedule-job

APIs for scheduling:

1. Non-Recurring Schedule: POST {host}:{port}/v1/schedule 

HEADERS
{"Authentication": "[Signed Bearer Token]"}

BODY
{  "schedule" : {
        "recurring" : "false",
        "frequency": { "date" : "(2016, 01, 31, 13, 24, 00)" }
    },
    "job": {
        "endpoint": "http://contour.myntra.com/contour/stories/publish",
        "method": "POST",  // supports GET, POST, PUT
        "payload": {
            "title": "Hello",
            "image": "http://www.someurl.com/image.jpg"
        } (optional),
        "headers": {
            "Content-Type": "application/json",
            "Accept": "application/json",
            "Authentication": "...",
        } (optional),
    },
    "id": "[unique job ID] (optional)" // preferably uuid
}
